**Завдання для попередньої підготовки:**

**Archiving (Архівування)** — об'єднання кількох файлів в один без обов'язкового стиснення для зручності зберігання.  
**Compression (Стиснення)** — зменшення розміру файлу за допомогою математичних алгоритмів.  
**Lossless (Без втрат)** — метод стиснення, що дозволяє повністю відновити оригінальні дані.  
**Verbose (Докладний)** — параметр (-v), який змушує команду виводити деталі процесу виконання.  
**Standard Output (stdout)** — стандартний потік виведення даних (зазвичай на екран).

* **Призначення команд Завдання 5.1**

| Команда | Опис | Основні параметри |
| :---- | :---- | :---- |
| **tar** | Створення та маніпуляція архівами. | \-c (create), \-x (extract), \-v (verbose), \-f (file), \-t (list). |
| **gzip** | Класичне швидке стиснення (алгоритм DEFLATE). | \-d (decompress), \-r (recursive), \-k (keep source), \-1..-9 (рівень). |
| **bzip2** | Вищий ступінь стиснення, ніж у gzip (Burrows-Wheeler). | \-d (decompress), \-z (compress), \-s (small memory mode). |
| **xz** | Найефективніше стиснення (LZMA2), але повільне. | \-d (decompress), \-e (extreme), \-0..-9 (рівень). |
| **zip** | Популярний формат для сумісності з Windows. | \-r (recursive), \-e (encrypt). |

**Встановлення:** У Ubuntu/Debian через sudo apt install \[назва\].

*  **Приклади архівації  Завдання 5.2**  
1. tar \-czvf backup.tar.gz /etc — створення gzipped архіву директорії /etc.  
2. tar \-cjvf docs.tar.bz2 \~/Documents — створення bzip2 архіву.  
3. tar \-cJvf images.tar.xz ./Pictures — створення xz архіву.

* **Робота з текстом Завдання 5.3**  
1. **cat:** Виводить весь вміст файлу на екран.  
2. **less / more:** Посторінковий перегляд великих файлів.  
3. **head / tail:** Виводять перші або останні 10 рядків файлу відповідно.  
   

* **Канали та фільтри Завдання 5.4**  
1. **Канали (Pipes |):** Передають вивід однієї команди на вхід іншої (наприклад, ls | grep .txt).  
2. **Потоки:** stdin (0), stdout (1), stderr (2).  
3. **Фільтри:** Команди, що обробляють текст (sort, uniq, grep, wc).

1\. Порівняльна характеристика процесів стискання та архівування

Стиснення та архівування часто використовуються разом, але виконують різні завдання:

| Характеристика | Стиснення | Архівування |
| :---- | :---- | :---- |
| Призначення | Зменшення розміру файлу шляхом видалення надлишкових даних | Об’єднання декількох файлів і каталогів в один файл без зміни їх змісту |
| Результат | Менший файл (наприклад, file.txt → file.txt.gz) | Один файл, що містить багато файлів (наприклад, archive.tar) |
| Алгоритми | gzip (DEFLATE), bzip2, xz, lzma | tar, cpio, ar (архівування без стискання) |
| Використання | Економія місця на диску, передача по мережі | Збереження структури каталогів, резервування даних |
| Команди Linux | gzip file, bzip2 file, xz file | tar \-cf archive.tar folder/ |
| Можливість об’єднання | Може комбінуватися з архівуванням (наприклад, tar \-czf) | Часто поєднується зі стисканням для економії місця |

Висновок:

* Архівування відповідає за зручність зберігання та структуру файлів.  
* Стиснення відповідає за економію дискового простору.  
* Найефективніше використовувати комбінацію архівування та стискання (наприклад, tar \+ gzip або tar \+ xz).

2\. Програми для стискання та архівування в Linux

Окрім стандартних tar, gzip, bzip2, xz, існує багато інших програм:

1. 7zip (p7zip)  
   * Формат: .7z  
   * Переваги: високий коефіцієнт стиснення, підтримка різних алгоритмів, кросплатформенність.  
   * Використання:

      7z a archive.7z folder/  
     7z x archive.7z

2. zip / unzip  
   * Формат: .zip  
   * Переваги: сумісність з Windows, швидке стиснення, підтримка шифрування.  
   * Використання:

      zip \-r archive.zip folder/  
     unzip archive.zip

3. rar / unrar  
   * Формат: .rar  
   * Переваги: висока ступінь стиснення, багатотомні архіви, відновлення пошкоджених архівів.  
   * Використання:

      rar a archive.rar folder/  
     unrar x archive.rar

4. zstd (Zstandard)  
   * Новий алгоритм, швидший за gzip та bzip2, з високою ефективністю.  
   * Використання:

      zstd file.txt  
     zstd \-d file.txt.zst

5. lzop  
   * Дуже швидкий алгоритм стиснення, але трохи гірший за gzip по коефіцієнту.  
   * Використання:

      lzop file.txt  
     lzop \-d file.txt.lzo

3\. Порівняння алгоритмів стискання в Linux

| Алгоритм | Програма | Швидкість | Коефіцієнт стиснення | Переваги | Недоліки |
| ----- | ----- | ----- | ----- | ----- | ----- |
| DEFLATE | gzip | Дуже швидкий | Середній | Швидке стиснення, сумісність | Не завжди найефективніше |
| Burrows-Wheeler | bzip2 | Повільний | Високий | Кращий коефіцієнт для текстових файлів | Повільне стискання та розпакування |
| LZMA | xz | Повільний | Дуже високий | Максимальна економія місця | Дуже повільно на великих файлах |
| Zstandard | zstd | Дуже швидкий | Високий | Баланс швидкості та стиснення | Новий формат, не завжди підтримується старим ПЗ |
| LZO | lzop | Надшвидкий | Середній | Для оперативного стискання, стрімінгу | Менший коефіцієнт |

Висновок:

* Найшвидший: lzop, gzip  
* Найефективніший за розміром: xz, bzip2  
* Оптимальний баланс: zstd

6\. Використання стискання та архівування для резервування даних

* Резервування (backup):  
  * Архівування дозволяє об’єднати всі файли проекту або каталогу в один файл, зменшуючи ризик пропущення файлів.

  * Приклади команд Linux:

     tar \-czf backup.tar.gz /home/user/project/

* Інші задачі системного адміністрування:  
  * Передача великих даних по мережі з меншою витратою трафіку.  
  * Збереження журналів та логів у стислому вигляді.  
  * Архівування старих даних для оптимізації дискового простору.

7\. Призначення директорії /dev/null

* /dev/null – це спеціальний файл у Linux, який відкидає всі записані в нього дані і повертає кінець файлу при читанні.  
* Призначення:  
  * Використовується для ігнорування виводу команд або помилок.  
  * Наприклад:

     command \> /dev/null   \# Ігнорує стандартний вивід  
    command 2\> /dev/null  \# Ігнорує помилки  
    command &\> /dev/null  \# Ігнорує все

* Застосування в адмініструванні:  
  * Тихе виконання скриптів без виводу на екран.  
  * Перенаправлення непотрібного виводу в лог-файли.
